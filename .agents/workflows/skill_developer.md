# 💻 [스킬] 개발자: React 상태 관리 및 컴포넌트 구조화

이 스킬은 **개발자 에이전트** 가 새로운 컴포넌트를 개발하거나 로직을 작성할 때 반드시 적용해야 할 기준표입니다.

## 📝 개발 워크플로우:
1. **정확한 타입(Type) 선언**: 
   - `any` 타입을 엄격히 금지합니다. 모든 데이터를 명확한 `interface`나 `type`으로 정의하세요.
2. **단일 책임 원칙 (SRP)**:
   - 하나의 파일/함수가 너무 방대해지지 않도록 작고 독립적인 단위로 분리합니다.
3. **최적화 및 메모이제이션**:
   - `useMemo`, `useCallback`을 적절히 활용하여 리렌더링을 최적화하세요.
4. **한글 주석 (Comments) 규칙**:
   - 핵심 로직 및 비즈니스 프로세스(예: 로또 번호 추천 알고리즘 등)가 포함된 코드 블럭 상단에는 반드시 한글로 상세한 설명 주석을 적습니다.

> **💡 예시 주석**: // 이 함수는 1부터 45까지의 숫자 중에서 가중치를 적용하여 추천 번호 6개를 반환합니다.
