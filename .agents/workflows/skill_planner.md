# 📊 [스킬] 기획자: 요구사항 분석 및 문서화

이 스킬은 **기획자 에이전트** 가 새로운 피처나 아이디어를 기획할 때 적용합니다.

## 📝 워크플로우 단계:
1. **사용자 요청 분석**: 
   - 유저의 대화 내용을 바탕으로 핵심 니즈를 파악합니다.
2. **해결책 제안**: 
   - 기술적 옵션과 UX를 여러 가지 각도로 제안합니다.
3. **Task 분할 (WBS)**:
   - 개발자와 디자이너가 작업하기 좋도록 기능을 단계별로 쪼갭니다.
4. **승인 요청 (Confirm)**: 
   - 실무를 진행하기 전 항상 사용자(프로젝트 오너)의 승인을 받습니다.

> **💡 중요**: 기획안 문서를 작성할 때는 항상 명확하고 친절한 한국어를 사용하세요.
